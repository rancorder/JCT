# review_checklist.md
有限会社ジェイシーティー様｜プレ商談・1次商談スライド 商談前レビューチェックリスト

本ファイルは内部運用資料（顧客提示物ではない）。商談前に話者本人、または確認者がこのチェックリストに沿って最終確認を行う。

---

## A. ポリシー適合チェック（最重要・妥協不可）

- [ ] 全スライド・全トークの中に「既存仕入先」を否定・軽視する表現がないか（例：「非効率」「古い」「見直すべき」等）
- [ ] 「安い海外調達」「安く仕入れる」といった安さ単独の訴求になっていないか
      →「補完調達ルート」「第2の調達ルート」「選択肢の追加」という表現で統一されているか
- [ ] コスト訴求がメリット4項目（供給リスク分散／納期安定／比較見積／選択肢拡大）の中で並列に扱われており、単独の見出しや最初の訴求点になっていないか
- [ ] スライド本体で「置き換え」「切り替え」「乗り換え」という言葉が使われていないか
- [ ] Slide 4は「置き換えではありません」ではなく、「今のお取引を活かしたまま、調達ルートをもう一つ持つ考え方」のような肯定形見出しになっているか
- [ ] サービス説明（Slide 5以降）より前に、相手の調達状況・リスクの整理（Slide 2〜3）が置かれているか

## B. 情報の正確性チェック

- [ ] 数値・実績（導入社数、具体的な削減率等）で、正本または関係者確認が取れていないものが記載されていないか
- [ ] 5〜10％削減の表現を使う場合、主訴求ではなくSlide 6の補足扱いになっているか
- [ ] decision_log.mdの「未確定・確認が必要な事項」がすべて解消されているか、未解消の場合はスライド上で断定表現になっていないか（「〜の場合があります」等の留保表現になっているか）
- [ ] 「対象になりやすい企業」（Slide 8）の適合条件が、実際にJCTが想定するターゲット像と一致しているか
- [ ] Slide 5には台本で確認済みのFACT（木材輸入25年以上／中国中心の海外工場ネットワーク／品質・価格・納期確認／サンプル確認・条件調整）が反映されているか
- [ ] 樹種・規格・最小ロット・標準リードタイム・対応可能商材一覧は、未確認の場合に断定されていないか

## C. トーン・言葉遣いチェック

- [ ] 相手の状況を一方的に決めつける表現（「〜で困っていますよね」等）がなく、伝聞・問いかけ形になっているか
- [ ] 「ご決断」「ご契約」等、即決を想起させる言葉がSlide 2・Slide 9・Slide 10で使われていないか
- [ ] 敬語・貴社／御社表記の統一（社内資料でも顧客提示物と同水準の表記ルールを適用する）

## D. 内部情報の混入チェック（顧客提示物としての適格性）

- [ ] slide_spec.md／storyboard.md／talk_script.mdの中に、内部評価・勝ち筋・スコアリング・AI作業メモ等の記述が残っていないか（decision_log.md／review_checklist.mdのみに限定されているか）
- [ ] 「確認事項」表記が、顧客に見せる想定のスライド本体（画面表示部分）ではなく、仕様書内の注記としてのみ存在しているか

## E. 構成・流れチェック

- [ ] Slide 1〜2で「これは提案の場ではない」ことが明確に伝わる構成になっているか
- [ ] Slide 3で相手が「特に困っていない」と回答した場合の受け止め方が用意されているか（talk_script.md参照）
- [ ] Slide 4（JCTの考え方）が、Slide 5（できること）より前に置かれているか
- [ ] Slide 9・10で、次回商談が「決定の場」ではなく「確認の場」として位置付けられているか
- [ ] 「代表同席」が、JCT側代表同席と相手側判断者同席で混同されていないか
- [ ] 相手側には「代表者様」ではなく「調達判断に関わる方」「社内確認に関わる方」と表現されているか
- [ ] 次回商談のクロージング先（2次商談トスアップ／見積取得／訪問／その他）が、商談前に事前確認され、Slide 10の話者ノートに反映されているか

## F. 商談当日の直前確認

- [ ] 対象企業名・日付がSlide 1に正しく反映されているか
- [ ] 画面共有時の表示崩れ（フォント・改行）がないか、事前に画面共有テストを実施したか
- [ ] 話者が「難所への対応」（talk_script.md末尾）を事前に読み込んでいるか
- [ ] 商談後にSlide 8での相手の回答内容をメモする準備（別紙・別ツール）があるか

---

## 最終判定

- [ ] 上記A〜Fすべてにチェックが入るまで、本資料を対外的に使用しない
- [ ] A（ポリシー適合）に1つでも未チェックがある場合は、他項目の状況にかかわらず使用を保留する
